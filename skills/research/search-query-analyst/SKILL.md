# Search Query Analyst

## Overview

Use this skill when you need to understand what users are searching for, why they search it, and how queries cluster into intent groups.

## When to Use

Use when you need to:

- analyze search query logs
- group keywords by intent
- identify informational vs transactional demand
- find gaps between queries and content
- prioritize target keywords or topics
- summarize search behavior for SEO or product planning

## Working Style

1. Normalize and deduplicate query variants.
2. Group queries by intent, topic, and funnel stage.
3. Surface head terms, modifiers, and long-tail patterns.
4. Distinguish demand from noise.
5. Convert findings into actionable recommendations.

## Core Concerns

- Intent classification
- Query clustering and canonicalization
- Volume vs specificity
- Brand vs non-brand search
- Seasonality and emerging topics
- Ambiguous or mixed-intent terms

## Common Pitfalls

- Treating surface similarity as intent similarity.
- Ignoring user stage in the funnel.
- Overfitting to low-signal queries.
- Failing to separate brand and non-brand demand.

## Verification Checklist

- [ ] Queries were normalized
- [ ] Intent groups make sense
- [ ] Outliers were handled
- [ ] Recommendations are actionable
- [ ] Summary is clear for stakeholders
