# Neural Substrate with RuVector

## Overview

Use this skill when the system needs a neural substrate integrated with RuVector.
It serves as a compact reference for memory-backed retrieval and agentic coordination.

## When to Use

Use when you need to:

- connect a neural substrate to RuVector
- support memory and retrieval from the same system
- coordinate agent behavior through shared neural infrastructure
- keep routing, recall, and learning aligned

Do not use for:

- standalone retrieval tasks without neural coordination
- simple documentation changes
- verdict evaluation or memory consolidation

## Neural Substrate

Purpose: integration with RuVector.

Typical responsibilities:

- provide shared memory access
- support pattern retrieval
- coordinate expert routing
- keep learned structure available across tasks

## Usage Notes

- Treat RuVector as the retrieval/memory backbone.
- Use the neural substrate to connect learning, recall, and dispatch.
- Keep the substrate lightweight and stable.
- Prefer shared representations over duplicated state.

## Common Pitfalls

1. Splitting neural state from retrieval state unnecessarily.
2. Using the substrate without consistent memory patterns.
3. Treating RuVector as an isolated database instead of a coordination layer.

## Verification Checklist

- [ ] RuVector integration is clear
- [ ] Memory and retrieval are linked
- [ ] Coordination path is defined
- [ ] Shared neural substrate remains stable
