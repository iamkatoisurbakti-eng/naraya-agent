# Claude Flow Worker Priorities

## Overview

Use this skill as a compact reference for worker priority levels and their meaning.
It is meant for quick lookup when configuring worker behavior.

## When to Use

Use when you need to:

- choose a worker priority
- distinguish deep learning behavior from performance tuning
- keep worker configuration simple and explicit

Do not use for:

- general workflow orchestration
- task execution planning
- unrelated coding tasks

## Worker Priority Table

| Worker Priority | Description |
|---|---|
| ultralearn | normal | Deep knowledge acquisition |
| optimize | high | Performance optimization |
| consolidate | low | Memory consolidation |

## Usage Notes

- Use ultralearn for knowledge-heavy or learning-oriented work.
- Use optimize for speed, efficiency, and performance-focused work.
- Keep the label and description aligned with the worker's intent.

## Common Pitfalls

1. Using optimize when the task is really about learning or discovery.
2. Using ultralearn when the task is mostly about speed tuning.
3. Treating worker priority as a workflow role.

## Verification Checklist

- [ ] Priority chosen correctly
- [ ] Description matches the intended behavior
- [ ] Worker purpose is clear
