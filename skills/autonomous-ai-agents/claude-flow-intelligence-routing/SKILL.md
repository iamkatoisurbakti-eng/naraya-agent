# Claude Flow Intelligence Routing

## Overview

Use this skill as a compact routing reference for advanced trigger-based worker behavior.
It maps trigger names to their primary purpose and notes the neural/performance concepts behind the routing layer.

## When to Use

Use when you need to:

- map trigger words to worker behavior
- route work based on task intent
- separate predictive loading from analysis, documentation, refactoring, and benchmarking
- reference the underlying neural optimization concepts

Do not use for:

- one-off tasks without trigger routing
- full multi-agent orchestration plans
- implementation of the routing engine itself

## Neural / Optimization Concepts

- SONA: Self-Optimizing Neural Architecture, sub-0.05ms adaptation
- MoE: Mixture of Experts for specialized routing
- HNSW: 150x-12,500x faster pattern search
- EWC++: Elastic Weight Consolidation, helps prevent forgetting
- Flash Attention: 2.49x-7.47x speedup

## Usage Notes

- Use predict and preload for early resource readiness.
- Use audit for high-priority security analysis.
- Use map and deepdive for codebase understanding and root-cause work.
- Use document when the output should become reusable docs.
- Use refactor after analysis when behavior must remain stable.
- Use benchmark and testgaps to validate quality and coverage.

## Common Pitfalls

1. Using predict or preload when the task actually needs audit or deepdive.
2. Treating benchmark as a replacement for test coverage analysis.
3. Using refactor before the codebase is mapped.
4. Forgetting that audit has the highest priority here.
