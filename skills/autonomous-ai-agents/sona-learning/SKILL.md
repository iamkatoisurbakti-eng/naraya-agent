# SONA Learning

## Overview

Use this skill when the system needs SONA-style self-optimizing learning.
It focuses on fast adaptation, pattern retention, and stable behavior updates.

## When to Use

Use when you need to:

- adapt behavior quickly from new outcomes
- retain useful patterns across runs
- optimize routing or worker selection over time
- keep learning lightweight and incremental

Do not use for:

- one-off execution without feedback
- raw retrieval tasks
- final verdict evaluation or memory consolidation

## SONA

Purpose: self-optimizing neural adaptation.

Typical responsibilities:

- update behavior from recent outcomes
- preserve useful patterns
- reduce response lag in repeated tasks
- support fast local improvement

## Usage Notes

- Use SONA when the system should improve after each run.
- Keep updates small and frequent.
- Pair SONA with retrieval and consolidation for stable learning.
- Prefer compact pattern storage over verbose logs.

## Common Pitfalls

1. Treating SONA as a replacement for judgment.
2. Updating too much at once.
3. Forgetting to preserve important patterns before adapting.

## Verification Checklist

- [ ] Learning signal identified
- [ ] Update was small and stable
- [ ] Useful pattern retained
- [ ] Behavior improved over time
