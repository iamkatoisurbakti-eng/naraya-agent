# ReasoningBank

## Overview

Use this skill when the system needs a ReasoningBank layer for storing, retrieving, and consolidating structured reasoning patterns.
It acts as a compact memory system for reusable inference and problem-solving traces.

## When to Use

Use when you need to:

- store structured reasoning patterns
- retrieve prior reasoning traces
- consolidate useful problem-solving sequences
- reuse learned decision paths across tasks

Do not use for:

- one-off answers with no reuse value
- raw document storage
- final verdict evaluation

## Core Functions

- store reasoning traces
- retrieve relevant reasoning patterns
- consolidate stable problem-solving rules
- support later decision reuse

## Usage Notes

- Keep entries small and structured.
- Capture the reasoning pattern, not just the final answer.
- Retrieve before inventing a new approach.
- Consolidate only patterns that proved useful.

## Common Pitfalls

1. Saving overly verbose traces with no reusable structure.
2. Retrieving too broadly and diluting relevance.
3. Consolidating untested reasoning patterns.
4. Treating ReasoningBank as a dump instead of a pattern store.

## Verification Checklist

- [ ] Reasoning pattern stored
- [ ] Relevant pattern retrieved when needed
- [ ] Stable patterns consolidated
- [ ] Memory remains structured and reusable
