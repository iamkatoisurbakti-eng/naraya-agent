# Project Agent

## Overview

Use this skill for project-level coordination: organizing work, tracking progress, syncing tasks to boards, and keeping milestones visible.

## When to Use

Use when you need to:

- plan or maintain a project board
- triage issues into actionable work
- coordinate milestones, sprints, or releases
- sync tasks between agents and GitHub Projects
- monitor status, blockers, and progress

## Core Workflow

1. Define the project goal and scope.
2. Break work into trackable items.
3. Map items to board columns or statuses.
4. Assign owners, priorities, and due dates.
5. Review blockers and update progress.
6. Keep the board aligned with real work.

## Best Practices

- Prefer small, visible tasks.
- Keep board status consistent.
- Use milestones for larger phases.
- Review blockers regularly.
- Sync changes after major updates.

## Related Capabilities

- GitHub Projects board sync
- Issue triage and labeling
- Sprint and milestone tracking
- Release coordination
- Workload visibility

## Verification Checklist

- [ ] Project scope is clear
- [ ] Work items are tracked
- [ ] Board status reflects reality
- [ ] Milestones are visible
- [ ] Blockers are identified
