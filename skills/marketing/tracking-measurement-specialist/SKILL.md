# Tracking & Measurement Specialist

## Overview

Use this skill when tracking and measurement must be reliable enough to guide decisions: analytics events, conversion attribution, pixels, tags, and reporting.

## When to Use

Use when you need to:

- set up or repair event tracking
- validate pixel or tag firing
- audit conversion measurement
- define funnel metrics and reporting
- compare platform vs source-of-truth data
- reduce tracking noise or blind spots

## Working Style

1. Define the key actions that must be measured.
2. Map events, properties, and destinations.
3. Verify implementation in-browser and in reporting.
4. Check for duplicates, gaps, and mismatches.
5. Align measurement with business goals.

## Core Concerns

- Event naming and schema consistency
- Tag/pixel firing accuracy
- Conversion definitions
- Attribution windows and deduplication
- Cross-domain or device continuity
- Source-of-truth reporting

## Common Pitfalls

- Tracking vanity events instead of decision-making events.
- Duplicated or missing conversion events.
- Mismatch between platform metrics and backend truth.
- No validation after deployment.

## Verification Checklist

- [ ] Key events are defined
- [ ] Tracking implementation was validated
- [ ] Duplicates and gaps were checked
- [ ] Reporting matches expected behavior
- [ ] Business metrics are measurable
