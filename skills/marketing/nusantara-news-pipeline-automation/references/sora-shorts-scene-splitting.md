# Sora Shorts scene splitting

Use this when the user wants one YouTube Short generated from multiple Sora prompts.

## Core rule
- Treat **1 prompt = 1 scene**.
- Default current newsroom Shorts target: **5 scenes × 4 seconds = ~20 seconds total**.
- Keep each scene around **4 seconds** when the user wants a tight news short.
- If the user asks for a shorter cut, stay within **15–20 seconds** total rather than drifting to 30 seconds.

## Recommended structure for 30s
- Scene 1: hook / opening
- Scene 2: context / build-up
- Scene 3: main event
- Scene 4: payoff / reaction
- Optional Scene 5–6: extra detail or closing line

## Prompt template per scene
- Subject
- Action
- Location
- Mood
- Camera movement
- Visual style
- Duration

Example:
> A young Indonesian creator walks into a neon-lit studio at night, holding a smartphone, confident expression, cinematic lighting, slow dolly-in, realistic style, shallow depth of field, 6 seconds.

## Consistency rules
- Keep the same character, wardrobe, lighting, and style across scenes.
- Avoid packing multiple events into one scene.
- Keep the camera language simple and repeatable.

## Pitfalls
- One huge prompt for a full 30s Short usually gets muddy.
- If a scene is too long, split it instead of adding more actions.
- The final video should be assembled in an editor; Sora scenes are the source clips.
