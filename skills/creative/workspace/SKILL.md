# Workspace

## Overview

Use this skill when the task is about managing a shared working area that may contain multiple artifacts, such as notes, mockups, layouts, references, or partial drafts.

## When to Use

Use when you need to:

- organize several related pieces in one place
- compare alternatives side by side
- move from rough structure to refined output
- keep a working area clear and navigable

Do not use for:

- single, isolated outputs with no need for arrangement
- backend-only tasks
- tasks where the final structure is already fixed

## Working Style

- Start by separating major sections.
- Keep the workspace legible and uncluttered.
- Make hierarchy obvious before refining details.
- Preserve room for iteration and comparison.

## Common Pitfalls

1. Overfilling the workspace too early.
2. Mixing unrelated ideas without structure.
3. Treating the workspace like a final deliverable too soon.

## Verification Checklist

- [ ] The workspace goal is clear
- [ ] Related items are grouped logically
- [ ] Alternatives can be compared easily
- [ ] The layout supports iteration
