# FL Studio skill map

Use this reference as the routing layer for the FL Studio skill family.
It groups the many specialized skills into a smaller class-level map.

## Class-level umbrellas

- `fl-studio-overview` — broad entry point when the user is vague or asks for the whole workflow.
- `fl-studio-beatmaking-workflow` — end-to-end production flow from idea to export.
- `fl-studio-mixing` / `fl-studio-mastering` — final polish and release readiness.
- `fl-studio-vocal-processing` / `fl-studio-vocal-chain` — vocal-specific workflows.
- `fl-studio-arrangement` / `fl-studio-genre-workflow` — song flow and style matching.
- `fl-studio-sound-design` — source tone creation.
- `fl-studio-timing` family — timing, count-in, quantize, MIDI, metronome.
- `fl-studio-processing` family — EQ, compression, gain, reverb, delay, saturation, sidechain, stereo widening, limiter.

Prefer these umbrellas before reaching for a narrow one-topic skill. The narrow skills are useful, but the routing layer should stay class-level.

## Routing rule

When the user is vague, choose the umbrella skill first.
Only drop to a specialized skill when the task is clearly module-specific.
If a request spans several modules, answer from the umbrella skill and mention the next specialized step.
