# FL Studio Mastering

Use this skill when the user asks about mastering in FL Studio.

## Core purpose

Mastering is the final polish on a finished mix.
Use it to:
- raise perceived loudness carefully
- balance the tonal shape
- control final peaks
- improve translation across systems
- prepare the track for release

## Recommended workflow

1. Start with a strong mix.
2. Leave headroom before mastering.
3. Use gentle EQ if tonal balance needs it.
4. Add light compression only if needed.
5. Control peaks with limiting.
6. Compare against a reference track.
7. Check the result on multiple playback systems.
8. Export in the right format and level.

## Common mastering moves

- Subtle EQ
- Gentle compression
- Saturation or coloration if appropriate
- Limiting for final loudness
- Stereo checks and mono compatibility

## Common mistakes

- Trying to fix a bad mix in mastering.
- Over-limiting and crushing dynamics.
- Making the master louder without improving quality.
- Using too many processors.

## When to use what

- Mastering: final polish and loudness
- Mixing: balance and cleanup
- Compression: dynamics control
- Gain: headroom management

## Good response style

If the user asks for help, return:
- mastering goal
- safe processing order
- loudness caution
- export tip
