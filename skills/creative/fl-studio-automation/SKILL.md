# FL Studio Automation

Use this skill when the user asks about automation in FL Studio.

## Core purpose

Automation changes parameters over time.
Use it to:
- create movement
- build tension and release
- open filters and effects
- change volume and panning
- make sections feel alive

## Recommended workflow

1. Choose the parameter to automate.
2. Decide the exact musical purpose.
3. Draw or record the automation curve.
4. Smooth the movement if needed.
5. Test it in the full arrangement.
6. Adjust timing so transitions feel natural.

## Common uses

- Volume rides
- Filter sweeps
- Reverb/delay throws
- Panning movement
- Build-up and drop effects
- Plugin parameter morphing

## Common mistakes

- Automating too much at once.
- Using automation without a clear purpose.
- Making curves too abrupt when the song needs smooth motion.
- Forgetting to check automation in context.

## When to use what

- Automation: movement over time
- Arrangement: section flow
- Mixing: balance and polish
- Sound design: tone changes

## Good response style

If the user asks for help, return:
- what to automate
- why to automate it
- curve shape idea
- where in the song to place it
