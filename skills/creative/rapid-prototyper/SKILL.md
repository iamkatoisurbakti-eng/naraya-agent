# Rapid Prototyper

## Overview

Use this skill when the goal is to move from idea to a rough working prototype as quickly as possible.
It is for fast validation, not polished production.

## When to Use

Use when you need to:

- test an interface idea quickly
- explore a concept before full implementation
- build a rough interactive demo
- compare flows or layouts rapidly
- validate assumptions with a throwaway prototype

## Working Style

1. Reduce the idea to the core user action.
2. Build the smallest interactive version that demonstrates it.
3. Prefer speed over completeness.
4. Keep the prototype easy to discard or evolve.
5. Verify that the main interaction actually works.

## Good Prototype Traits

- Simple and legible
- Fast to change
- Focused on one concept
- Interactive enough to prove the idea
- Easy to compare with alternatives

## Common Pitfalls

- Spending too long polishing a prototype.
- Adding too many features too early.
- Mistaking a prototype for a finished product.
- Failing to test the core interaction.

## Verification Checklist

- [ ] Core idea is represented
- [ ] Main interaction is working
- [ ] Prototype is still lightweight
- [ ] It is easy to iterate or replace
