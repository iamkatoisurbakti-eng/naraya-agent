# Incident Response Commander

## Overview

Use this skill when an incident needs clear command: security breach, service outage, data issue, or urgent operational failure.
It focuses on containment, triage, communication, recovery, and post-incident follow-up.

## When to Use

Use when you need to:

- coordinate an active incident
- contain security or availability impact
- triage symptoms and prioritize response actions
- assign responders and track status
- communicate progress and recovery steps
- prepare a postmortem or remediation plan

## Working Style

1. Stabilize first; reduce further damage.
2. Identify scope, impact, and blast radius.
3. Assign owners for containment, diagnosis, and communication.
4. Preserve evidence and logs where relevant.
5. Restore service, then confirm recovery.
6. Capture lessons and follow-up actions.

## Incident Phases

- Detect
- Triage
- Contain
- Eradicate
- Recover
- Review

## Common Pitfalls

- Debugging before stopping the bleed.
- Mixing incident command with long-term cleanup.
- Failing to keep stakeholders updated.
- Losing logs, evidence, or timelines.
- Declaring success before verification.

## Verification Checklist

- [ ] Impact is understood
- [ ] Containment steps are underway
- [ ] Roles and owners are assigned
- [ ] Communication is active
- [ ] Recovery is verified
- [ ] Follow-up actions are recorded
