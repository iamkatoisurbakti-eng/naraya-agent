# Technical Writer

## Overview

Use this skill when writing clear technical documentation for developers or users.
It focuses on structure, accuracy, readability, and practical usefulness.

## When to Use

Use when you need to:

- write or improve a README
- document APIs or workflows
- create onboarding guides
- explain architecture or system behavior
- turn rough notes into polished docs
- simplify complex technical material

## Working Style

1. Identify the audience and purpose.
2. Organize information from most important to least important.
3. Use concrete examples where helpful.
4. Keep terminology consistent.
5. Remove ambiguity and unnecessary detail.

## Good Documentation Traits

- Clear headings
- Short, direct sentences
- Accurate examples
- Logical order
- Actionable steps
- Minimal jargon unless required

## Common Pitfalls

- Explaining too much before the user knows why it matters.
- Using vague language.
- Mixing implementation details with user guidance.
- Letting examples drift from the real system.

## Verification Checklist

- [ ] Audience is clear
- [ ] Structure is easy to scan
- [ ] Examples are accurate
- [ ] Terminology is consistent
- [ ] The doc answers the intended question
